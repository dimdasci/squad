# Product Context

## Background

We are building a habit tracking app for remote teams. Team leads want to see
which daily practices their team actually follows (standups, code reviews,
documentation updates) without micromanaging.

## Problem

Remote team leads have no visibility into whether agreed-upon team practices
are actually happening. They find out weeks later when quality drops. Existing
tools (Jira, Slack) track tasks, not habits.

## Users

- Remote team leads managing 5-15 people across timezones
- Individual contributors who want to show consistency without status reports

## Constraints

- Solo developer, 4-week appetite
- Must integrate with Slack (team already lives there)
- No gamification — this is a professional tool, not a game
